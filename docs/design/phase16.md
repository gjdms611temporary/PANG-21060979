# Phase 16 설계 — Mission 1 난이도 튜닝 및 마무리

> 대상: [docs/PLAN.md](../PLAN.md) Phase 16
> 참고 기획: [docs/FEATURES/mission1.md](../FEATURES/mission1.md)

## 1. 범위 확인

**이번 Phase에서 만드는 것**
- 지금까지 임의 값으로 두었던 속도/개수/배치 등 상수를 Mission 1 기획에 맞게 조정
- 배경 테마 적용
- 전체 플로우 최종 점검 (버그/어색한 흐름 수정 위주 — 새 기능 추가 없음)

**이번 Phase는 새 기능을 만들지 않는다.** 그동안 임시(placeholder)로 두었던 값과 비주얼을 다듬는 마무리 단계다.

---

## 2. 조정 대상 상수 (`game/constants.ts`)

| 상수 | 이번 Phase 전 (임시값) | 조정 방향 |
|---|---|---|
| `BALLOON_SPEED_X` | Phase 5~9 구현 중 임의값 | 시리즈 내 가장 느린 수준으로 하향 |
| `GRAVITY` / 초기 낙하 속도 | Phase 6 구현 중 임의값 | 바운스 높이가 낮아지도록 하향 (플레이어 반응 여유 확보) |
| 등장 풍선 개수/배치 | Phase 5에서 1~2개 임의 배치 | 대(大) 풍선 1~2개로 확정 (기획 문서 기준) |
| `PLAYER_SPEED`, `WIRE_SPEED` | Phase 2~3 구현 중 임의값 | Mission 1 튜토리얼 성격에 맞게 필요 시 조정 |
| `SCORE_TABLE` | Phase 14에서 제안한 예시값 | 검토 피드백 반영해 확정 |

- 값 조정은 모두 `game/constants.ts` 한 파일 수정으로 처리 가능하도록 이미 Phase 2~9에서 상수를 분리해둔 구조를 그대로 활용한다 (로직 코드 변경 없음).

## 3. 배경 테마 적용

- Mission 1 배경: 도시/랜드마크 테마 정적 이미지
- Canvas 렌더링 순서 최상단(첫 draw)에 `ctx.drawImage(backgroundImage, 0, 0, CANVAS_WIDTH, CANVAS_HEIGHT)` 추가
- 이미지 에셋은 `src/assets/`에 배치 (예: `src/assets/mission1-bg.png`) — 실제 이미지 파일은 별도 준비 필요 (에셋 확보 방법은 검토 시 논의)

## 4. 최종 점검 체크리스트

- [ ] 메인 화면 → Mission 1 → 클리어/게임 오버 → 메인 화면, 전체 흐름이 끊김 없이 동작
- [ ] 이동/발사/회피 3가지 기본 조작만으로 클리어가 가능한지 (보조 무기/아이템 없음, PRD 4절 기준)
- [ ] 튜토리얼로서 난이도가 지나치게 어렵거나 쉽지 않은지 (실제 플레이 테스트로 판단)
- [ ] 콘솔 에러/경고 없이 정상 동작하는지

---

## 5. 확인 방법

1. 전체 플로우(메인 → Mission 1 → 클리어/오버 → 메인)를 처음 접하는 사람 기준으로 플레이해보고 위 체크리스트 항목을 확인
2. 배경 테마가 적용된 화면에서도 캐릭터/풍선/Wire가 잘 보이는지(대비/가독성) 확인
3. 조정된 속도값들이 "느리다"고 느껴지는지, 반응 여유가 충분한지 체감 평가

---

## 6. 검토 요청 사항

- 배경 이미지 에셋을 직접 준비해서 전달해주실지, 임시 플레이스홀더(단색/그라디언트)로 진행할지 확인 부탁드립니다.
- 상수 조정은 감각적 판단이 필요한 부분이 많아, 1차 조정 후 실제 플레이해보시고 추가 피드백(더 느리게/빠르게 등)을 주시면 반영하겠습니다.
