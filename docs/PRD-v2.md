# PANG (팡) 게임 — PRD v2 (Block 장애물 + 비주얼/UX 개선)

> 전제: [PRD.md](PRD.md)에 정의된 Mission 1 기본 규칙은 그대로 유지한다. 본 문서는 v1 구현(PLAN.md Phase 1~16) 완료 후 발견된 보완 사항을 정의한다.

## 1. 개요

v1 구현 리뷰 결과, 다음 두 가지가 보완되어야 한다.

1. **Block(벽) 장애물 누락**: 원작 PANG(1989)에는 화면 내 고정된 벽(Block)이 존재하여 Wire/풍선/플레이어의 이동을 막는 장애물 역할을 한다. v1에는 이 요소가 빠져 있었다. (참고: [FEATURES/game_rule.md](FEATURES/game_rule.md) 3.2절에 "바닥/장애물에 닿아 튕길 때마다"라는 문구가 이미 있었으나, "장애물(Block)"이 실제로 정의/구현되지 않았다.)
2. **비주얼/UX 디자인 미흡**: 현재 플레이어/풍선/Wire는 사각형·원 도형 placeholder이고, 배경은 단색 그라디언트다. 화면 구성(HUD, 메뉴, 클리어/게임오버 화면)도 최소 기능 수준에 머물러 있다.

## 2. Block(벽) 장애물

### 2.1 정의
- 화면 내 고정된 위치에 배치되는 사각형 벽. 스스로 움직이지 않는다.
- Wire, 풍선, 플레이어 모두 Block을 통과할 수 없다.

### 2.2 상호작용 규칙
- **Wire vs Block**: Wire가 Block에 닿으면 풍선을 맞히지 못한 채 즉시 소멸한다(천장에 닿았을 때와 동일하게 재발사 가능 상태로 전환).
- **풍선 vs Block**: 풍선이 Block에 부딪히면 화면 벽에 부딪혔을 때와 동일하게 진행 방향이 반전된다(수평 방향은 좌/우 반전, 위/아래에서 부딪히면 상하 반전).
- **플레이어 vs Block**: 플레이어는 Block을 통과할 수 없다 — 화면 좌/우 벽과 동일하게 이동이 막힌다.

### 2.3 배치
- Mission 1은 튜토리얼 성격을 유지하므로 Block 개수는 소수(1~2개 수준)로 제한한다.
- 정확한 개수/좌표/크기는 구현 단계에서 확정하고, 확정 시 검토를 받는다.
- Mission 1의 Block은 고정(정지) 상태로 한정한다.

### 2.4 확장 여지 (후속 미션)
- 후속 스테이지(Mission 2 이상)에서는 Block이 움직이는(예: 정해진 경로를 왕복 이동) 형태로 확장될 수 있다.
- Mission 1 구현 시에는 이 확장을 염두에 두고 Block의 위치를 고정값이 아닌 상태(state)로 다루되, 실제 이동 로직은 이번 범위에 포함하지 않는다.

## 3. 비주얼 디자인 개선

- 플레이어/풍선/Wire/Block을 현재의 사각형·원 도형 placeholder에서 실제 일러스트/스프라이트 수준으로 교체한다.
- 배경을 현재의 단색 그라디언트 placeholder에서 [FEATURES/mission1.md](FEATURES/mission1.md) 5절에 명시된 도시/랜드마크 테마 아트로 교체한다.
- 위 요소들의 스타일(색감, 선 굵기, 만화풍/픽셀풍 등 톤 앤 매너)은 후속 상세 기획에서 확정한다.

## 4. UX/화면 레이아웃 개선

- 메인 화면, 조작 방법 화면, 클리어/게임 오버 화면의 버튼과 텍스트 배치에 통일된 디자인 시스템(폰트, 색상, 여백)을 적용한다.
- HUD(점수/목숨)를 포함한 모든 인게임 UI 요소는 캔버스가 레터박스로 축소/확대되는 어떤 화면 비율에서도 항상 캔버스 내부에 위치해야 한다(v1에서 있었던 "HUD가 캔버스 밖에 그려지는" 회귀를 재발하지 않도록 설계 단계에서 명시).

## 5. 범위 및 우선순위

1. **1차 (기능)**: Block 장애물의 게임 로직/충돌 판정 추가
2. **2차 (비주얼)**: 아트 리소스 확보 및 적용
3. **3차 (UX)**: 화면 레이아웃/디자인 시스템 정비
